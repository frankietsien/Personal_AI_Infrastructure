# PLANS

`PLANS/` stores forward-looking planning artifacts — multi-step roadmaps, staged release plans, project phase breakdowns, and any document whose purpose is to describe intended future work rather than record past work. PRDs that span multiple sessions often live here, separate from any single Algorithm slug in `WORK/`.

This separation matters because plans are durable across sessions while `WORK/` slugs are scoped to single runs. A plan in `PLANS/` may be referenced by many `WORK/` slugs as the work is executed in stages.

Empty in fresh installs. Populates when the user or a planning skill explicitly creates a multi-session plan. Edit freely — plans are living documents.
